# DiskScope Design System

## Design Philosophy

Information First.

Every visual element must improve understanding.

Decorative elements without functional value should be avoided.

---

# User Goal

The application should help users identify storage problems as quickly as possible.

The interface should minimize cognitive load.

---

# Visual Principles

- High information density
- Clear hierarchy
- Immediate readability
- Consistent spacing
- Minimal visual noise

---

# Motion

Animations should communicate state changes.

Never use animation for decoration.

---

# Colors

Colors communicate meaning.

Green

Healthy

Yellow

Review

Red

Attention

---

# Typography

Readable before beautiful.

Information should always have higher priority than aesthetics.

---

# Icons

Icons support labels.

Icons never replace labels.

---

# Charts

Charts must simplify understanding.

Charts should never exist purely for visual appeal.

---

# Empty States

Always explain what happened.

Never leave the user wondering why nothing is displayed.
